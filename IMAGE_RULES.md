# IMAGE_RULES.md

오박사만능인테리어 이미지 처리 규칙

---

## 01 upload_images

- 작업 시작 시 반드시 `G:\OneDrive\01_울산오박사인테리어\obaksa_site\upload_images`를 확인한다.
- 이미지가 존재하면 신규 시공사례 후보로 간주한다.
- 자동 수행 순서는 이미지 확인, 신규 폴더 생성, `original` 폴더 생성, 원본 복사, 웹용 이미지 생성, `thumb` 생성이다.

---

## 02 폴더 및 파일명

- 새 시공사례 이미지는 `images/cases/[YYYY-MM-DD-영문작업명]/`에 저장한다.
- 폴더명과 파일명에는 한글, 공백, 대문자를 사용하지 않는다.
- 허용 문자는 영문 소문자, 숫자, 하이픈이다.
- 표준 파일명은 `thumb.jpg`, `before-01.jpg`, `before-02.jpg`, `after-01.jpg`, `after-02.jpg`, `instagram-thumb.jpg`, `webtoon-8cut.jpg`다.

---

## 03 이미지 최적화

- 리스트 썸네일은 `200KB ~ 400KB`를 권장한다.
- 일반 시공 사진은 `300KB ~ 800KB`를 권장한다.
- 메인 히어로/대표 이미지는 `800KB ~ 1.5MB`를 권장한다.
- 웹툰형 세로 장형 이미지는 `1MB ~ 2MB`를 권장한다.
- 파일명 정리, 경로 정리, 존재 여부 검증은 수행할 수 있다.
- 별도 압축 도구가 없으면 직접 압축이나 리사이징은 수행하지 말고 용량 초과 여부를 보고한다.
- 이미지가 깨지면 폴더명, 파일명, 확장자, 대소문자, 상대 경로, 실제 존재 여부를 순서대로 확인한다.

---

## 04 임시 파일

- 이미지 처리 중 실수로 생성된 임시 파일은 Git commit에 포함하지 않는다.
- `*_mistake.*`, `original_file_mistake.*`, `*.tmp`, `*.bak` 패턴은 임시 파일로 간주한다.
- 임시 파일 삭제가 필요하면 먼저 사용자 승인을 받는다.

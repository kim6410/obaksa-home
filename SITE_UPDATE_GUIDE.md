\# SITE\_UPDATE\_GUIDE.md



\# 오박사만능인테리어 홈페이지 수정 및 시공사례 추가 작업 규칙서



이 문서는 오박사만능인테리어 홈페이지를 수정하거나 새 시공사례를 추가할 때 반드시 먼저 읽어야 하는 기준 문서입니다.



작업자는 HTML, CSS, JS, 이미지, 배포 ZIP을 수정하기 전에 이 문서를 먼저 확인해야 합니다.



이 문서의 목적은 기존에 잘 적용된 화면 구성을 깨뜨리지 않고, 새 게시물 추가와 모바일 최적화, 배포 압축 작업을 일관되게 유지하는 것입니다.



\---



\# 1. 프로젝트 목적



오박사만능인테리어 홈페이지는 울산 지역 기반 집수리, 인테리어, 전기수리, 누수복구, 방화문수리, 욕실수리, 도어락, 조명교체 등 생활밀착형 시공 사례를 소개하는 정적 HTML 홈페이지입니다.



핵심 목적은 아래와 같습니다.



1\. 울산 지역 고객이 실제 시공사례를 보고 신뢰를 느끼게 한다.



2\. 블로그, 인스타그램, 네이버 플레이스 등 외부 콘텐츠와 홈페이지를 연결한다.



3\. 모바일 방문자가 빠르게 시공사례를 확인하고 문의할 수 있게 한다.



4\. 오박사만능인테리어의 현장감, 친근함, 30년 경력의 실무 신뢰감을 보여준다.



5\. GitHub Pages 또는 정적 호스팅 환경에서 가볍고 안정적으로 운영되도록 유지한다.



홈페이지는 단순 포트폴리오가 아니라 지역 고객의 문의 전환을 위한 영업 자산입니다.



따라서 디자인 수정은 예쁘게 보이는 것보다 모바일 가독성, 신뢰감, 빠른 접근성, 문의 유도에 우선순위를 둡니다.

\# 2. 폴더 구조 설명



프로젝트 기준 경로는 다음과 같습니다.



```text

G:\\OneDrive\\01\_울산오박사인테리어\\obaksa\_site\\obaksa-home

```



상위 작업 경로는 다음과 같습니다.



```text

G:\\OneDrive\\01\_울산오박사인테리어\\obaksa\_site

```



주요 구조는 아래와 같습니다.



```text

obaksa-home/

├─ index.html

├─ about.html

├─ cases.html

├─ community.html

├─ contact.html

├─ assets/

│  ├─ css/

│  └─ js/

├─ images/

│  ├─ cases/

│  └─ gallery/

├─ make-release.ps1

└─ SITE\_UPDATE\_GUIDE.md

```



각 파일과 폴더의 역할은 아래와 같습니다.



\## index.html



홈페이지 메인 화면입니다.



최근 시공사례 1개가 반드시 노출되어야 합니다.



모바일 아코디언 메뉴가 포함되어 있는 경우, 제목 옆에 펼침 표시가 있어야 합니다.



\## cases.html



전체 시공사례 목록 페이지입니다.



카드형 미리보기는 최근 시공사례 2개만 노출합니다.



아래쪽 목록에는 전체 시공사례가 리스트 형태로 표시됩니다.



\## about.html



오박사만능인테리어 소개 페이지입니다.



업체 소개, 신뢰 요소, 현장 경험, 서비스 철학이 들어갑니다.



\## community.html



커뮤니티 또는 외부 콘텐츠 연결 페이지입니다.



블로그, 인스타그램, 소식 콘텐츠와 연결될 수 있습니다.



\## contact.html



문의 안내 페이지입니다.



전화, 위치, 상담 안내, 외부 링크가 들어갑니다.



\## assets/css/



전체 디자인, 반응형 레이아웃, 모바일 스타일을 관리합니다.



모바일 목록, 버튼, 아코디언 스타일을 수정할 때 반드시 확인해야 합니다.



\## assets/js/



아코디언, 슬라이드, 인터랙션 등이 들어갈 수 있습니다.



아코디언 열림/닫힘 표시가 JS로 제어되는 경우 이 폴더를 확인합니다.



\## images/cases/



시공사례별 이미지 폴더입니다.



새 시공사례를 추가할 때는 날짜와 작업 내용을 알 수 있는 폴더명을 사용합니다.



예시:



```text

images/cases/2026-06-07-led-module/

images/cases/2026-06-06-door-repair/

images/cases/2026-06-05-sink-collapse/

```



\## images/gallery/



기존 갤러리 이미지 폴더입니다.



수정용 ZIP에는 포함하지 않는 것을 기본 원칙으로 합니다.



홈페이지 운영에는 필요할 수 있으므로 함부로 삭제하지 않습니다.



\## make-release.ps1



배포용 ZIP을 만드는 PowerShell 스크립트입니다.



GitHub Pages 또는 배포용 파일을 만들 때 사용합니다.



\## make-chatgpt-package.bat



ChatGPT에게 전달할 수정용 ZIP을 만드는 배치 파일입니다.



상위 폴더인 아래 경로에 둘 수 있습니다.



```text

G:\\OneDrive\\01\_울산오박사인테리어\\obaksa\_site

```



수정용 ZIP은 전체 홈페이지가 아니라 수정에 필요한 파일만 포함해야 합니다.

\# 3. 수정 전 반드시 확인할 파일



작업 전에는 반드시 아래 파일을 확인합니다.



```text

SITE\_UPDATE\_GUIDE.md

index.html

cases.html

assets/css/

assets/js/

images/cases/

```



새 시공사례를 추가하는 경우에는 반드시 아래 항목을 확인합니다.



1\. 기존 cases.html의 시공사례 카드 구조



2\. 기존 cases.html의 전체 목록 구조



3\. index.html의 최근 시공사례 1개 노출 영역



4\. 모바일에서 목록 버튼이 작게 표시되는지



5\. 카드형 미리보기 버튼명이 `상세보기`, `Instagram`인지



6\. 새 이미지 폴더명과 이미지 경로가 실제로 맞는지



7\. 블로그 URL과 인스타그램 URL이 정확한지



작업 전에는 기존에 잘 작동하던 레이아웃을 임의로 갈아엎지 않습니다.



기존 CSS 클래스명을 최대한 유지하고, 필요한 경우에만 보조 클래스를 추가합니다.

\# 4. 새 시공사례 추가 규칙



새 시공사례를 추가할 때는 반드시 `cases.html`과 `index.html`을 함께 수정합니다.



\## 4-1. cases.html 수정 규칙



새 시공사례는 전체 시공사례 목록에 추가합니다.



목록에서는 아래 정보가 포함되어야 합니다.



```text

날짜

분류

제목

요약 설명

상세보기 또는 보기 버튼

블로그 링크

Instagram 링크가 있는 경우 Instagram 버튼

대표 이미지

```



날짜 표기는 사용자가 요청한 날짜를 기준으로 합니다.



예시:



```text

2026.06.07

6/7

```



홈페이지 내부 표현 방식이 기존에 통일되어 있으면 기존 형식을 따릅니다.



\## 4-2. 카드형 미리보기 규칙



cases.html 상단 또는 중간의 카드형 미리보기는 최근 게시물 2개만 출력합니다.



3개 이상 출력하지 않습니다.



새 게시물을 추가하면 가장 최신 게시물이 1번 카드가 되고, 기존 최신 게시물이 2번 카드로 밀립니다.



3번째 이후 게시물은 카드형 미리보기에서 제외하고 전체 목록에만 유지합니다.



\## 4-3. index.html 최근 시공사례 규칙



index.html에는 최근 시공사례 1개가 반드시 노출되어야 합니다.



새 시공사례를 추가하면 index.html의 최근 시공사례 영역도 최신 게시물로 갱신합니다.



index.html 최근 시공사례에는 아래 요소가 포함되어야 합니다.



```text

대표 이미지

날짜

분류

제목

짧은 요약

상세보기 버튼

Instagram 버튼이 있는 경우 Instagram 버튼

```



\## 4-4. 링크 규칙



블로그 링크가 있는 경우 `상세보기` 버튼에 연결합니다.



인스타그램 릴스 링크가 있는 경우 `Instagram` 버튼에 연결합니다.



버튼명은 반드시 아래처럼 유지합니다.



```text

상세보기

Instagram

```



`자세히보기`, `인스타릴스`, `릴스보기`, `보기` 등으로 바꾸지 않습니다.



\## 4-5. 제목 작성 규칙



시공사례 제목은 지역명 + 문제 상황 + 해결 방식이 드러나야 합니다.



예시:



```text

울산 북구 매곡동 에일린의뜰 전등 깜빡임 LED 모듈 교체

울산 북구 신천동 세면대 파손 긴다리 세면대 교체

울산 방화문 수리, 닫히지 않던 문을 안전하게 복구

```



너무 짧거나 광고성 문구만 있는 제목은 피합니다.

\# 5. 이미지 추가 규칙



이미지는 홈페이지 용량을 가장 크게 늘리는 요소입니다.



따라서 원본 이미지를 그대로 누적하지 않습니다.



\## 5-1. 이미지 폴더명 규칙



새 시공사례 이미지는 `images/cases/` 아래에 날짜와 작업 내용을 알 수 있는 폴더로 저장합니다.



예시:



```text

images/cases/2026-06-07-led-module/

images/cases/2026-06-06-door-repair/

images/cases/2026-06-05-sink-collapse/

```



폴더명은 영어 소문자, 숫자, 하이픈을 사용합니다.



한글 폴더명은 가능하면 사용하지 않습니다.



공백은 사용하지 않습니다.



\## 5-2. 이미지 파일명 규칙



이미지 파일명은 간단하고 순서가 보이게 작성합니다.



예시:



```text

thumb.jpg

before-01.jpg

before-02.jpg

after-01.jpg

after-02.jpg

instagram-thumb.jpg

webtoon-8cut.jpg

```



파일명에 공백을 넣지 않습니다.



카카오톡 원본 파일명처럼 긴 이름은 가능하면 정리합니다.



\## 5-3. 이미지 용량 권장 기준



홈페이지에 올리는 이미지는 압축본을 사용합니다.



권장 용량은 아래와 같습니다.



```text

썸네일 이미지: 200KB \~ 400KB

일반 시공 사진: 300KB \~ 800KB

큰 대표 이미지: 800KB \~ 1.5MB

웹툰형 긴 이미지: 1MB \~ 2MB

```



원본 사진은 OneDrive에 보관하고, 홈페이지에는 압축본만 넣는 것을 원칙으로 합니다.



\## 5-4. images/gallery 규칙



`images/gallery` 폴더는 수정용 ZIP에 포함하지 않습니다.



단, 실제 홈페이지에서 사용 중일 수 있으므로 삭제하지 않습니다.



수정용 ZIP에서는 제외하지만 운영 파일로는 유지합니다.



\## 5-5. 수정용 ZIP 이미지 포함 규칙



ChatGPT에게 전달하는 수정용 ZIP에는 `images/cases` 안의 최근 시공사례 이미지 폴더 1\~2개만 포함합니다.



전체 `images/gallery`나 오래된 시공사례 이미지 전체를 포함하지 않습니다.



기준은 아래와 같습니다.



```text

최근 시공사례 1개만 수정할 때: 최신 이미지 폴더 1개

최근 카드 2개 비교가 필요할 때: 최신 이미지 폴더 2개

전체 갤러리 수정이 필요할 때만 images/gallery 별도 포함

```



\## 5-6. 이미지 경로 검증 규칙



HTML에서 이미지 경로를 수정한 뒤에는 반드시 실제 파일이 존재하는지 확인합니다.



이미지가 깨지는 경우 대부분 아래 문제입니다.



```text

폴더명 오타

파일명 오타

대소문자 불일치

확장자 jpg/png/webp 불일치

상대경로 오류

```

\# 6. 모바일 레이아웃 규칙



오박사만능인테리어 홈페이지는 모바일 방문자가 많을 것을 기준으로 합니다.



따라서 모바일 화면에서는 정보가 높이를 너무 많이 차지하지 않도록 해야 합니다.



\## 6-1. 시공사례 목록 모바일 규칙



모바일에서 시공사례 목록 한 건은 최대한 컴팩트하게 표시합니다.



`보기` 버튼이 한 줄 전체를 차지하면 안 됩니다.



버튼은 날짜 옆에 작은 pill 버튼 형태로 배치합니다.



분류는 날짜 위쪽에 배치해 목록 높이를 줄입니다.



목록 한 건당 정보 구조는 아래처럼 유지합니다.



```text

분류

날짜 + 작은 버튼

제목

짧은 요약

```



\## 6-2. 카드형 미리보기 모바일 규칙



모바일에서도 카드형 미리보기는 최근 2개만 노출합니다.



카드 하나가 너무 길어지지 않도록 버튼 크기와 여백을 줄입니다.



버튼은 가능한 한 한 줄에 배치합니다.



공간이 부족하면 자연스럽게 줄바꿈은 가능하지만 버튼이 과도하게 커지면 안 됩니다.



\## 6-3. 아코디언 메뉴 모바일 규칙



모바일 index.html의 아코디언 메뉴는 누를 수 있다는 표시가 반드시 있어야 합니다.



제목 옆에는 기본 상태에서 아래 화살표를 표시합니다.



```text

▼

```



펼쳐진 상태에서는 위 화살표를 표시합니다.



```text

▲

```



사용자가 “여기를 눌러볼 수 있겠다”라고 직관적으로 느껴야 합니다.



\## 6-4. 아코디언 색상 규칙



아코디언 제목바의 기존 색상은 유지합니다.



펼쳐진 내용 영역은 제목바와 구분되도록 약간 밝은 청색 톤으로 표시합니다.



단, 너무 튀는 파란색은 피합니다.



고급스럽고 신뢰감 있는 밝은 청색 계열을 사용합니다.



예시:



```text

제목바: 기존 색상 유지

펼친 내용: 밝은 청색 배경 또는 은은한 청색 그라데이션

텍스트: 가독성 높은 짙은 색

```



\## 6-5. 모바일 여백 규칙



모바일에서는 좌우 여백이 과도하게 크면 안 됩니다.



이미지나 카드가 너무 작아 보이면 안 됩니다.



가독성과 화면 밀도를 함께 고려합니다.



버튼, 카드, 제목의 세로 간격은 지나치게 넓지 않게 유지합니다.



\---



\# 7. 버튼명 규칙



버튼명은 짧고 명확하게 유지합니다.



\## 7-1. 시공사례 카드 버튼



블로그 또는 상세 콘텐츠 링크 버튼명은 아래로 고정합니다.



```text

상세보기

```



인스타그램 릴스 링크 버튼명은 아래로 고정합니다.



```text

Instagram

```



\## 7-2. 사용하지 말아야 할 버튼명



아래 표현은 사용하지 않습니다.



```text

자세히보기

자세히 보기

인스타릴스

인스타 릴스 보기

릴스보기

블로그 보기

더보기

보기

```



단, 모바일 목록에서 아주 작은 pill 버튼이 필요한 경우에만 `보기`를 유지할 수 있습니다.



카드형 미리보기에서는 반드시 `상세보기`, `Instagram`을 사용합니다.



\## 7-3. 버튼 배치 규칙



PC 기준 카드형 미리보기에서는 `상세보기`와 `Instagram` 버튼이 한 줄에 나란히 보이도록 합니다.



모바일에서는 화면 폭에 따라 줄바꿈이 가능하지만 버튼이 지나치게 크거나 한 줄 전체를 차지하지 않도록 합니다.



\## 7-4. 버튼 색상 규칙



버튼 텍스트는 항상 잘 보여야 합니다.



버튼 배경이 진하면 텍스트는 흰색을 유지합니다.



호버 시에도 텍스트가 사라지거나 어두워지면 안 됩니다.

\# 8. 배포 ZIP 생성 규칙



배포용 ZIP과 ChatGPT 수정용 ZIP은 목적이 다릅니다.



두 파일을 혼동하지 않습니다.



\## 8-1. 배포용 ZIP



배포용 ZIP은 실제 GitHub Pages 또는 호스팅에 올릴 파일입니다.



`make-release.ps1`로 생성합니다.



배포용 ZIP에는 홈페이지 작동에 필요한 파일이 들어가야 합니다.



하지만 아래 항목은 절대 포함하지 않습니다.



```text

.git/

.gitignore

.vscode/

node\_modules/

\*.bak

\*.tmp

\*.log

\_\_MACOSX/

.DS\_Store

기존 release zip

```



배포용 ZIP 파일명 형식은 아래를 따릅니다.



```text

obaksa-home\_release\_YYYYMMDD\_HHMM.zip

```



\## 8-2. ChatGPT 수정용 ZIP



ChatGPT 수정용 ZIP은 수정에 필요한 최소 파일만 담는 압축 파일입니다.



상위 폴더에서 `make-chatgpt-package.bat`를 실행해 생성합니다.



기준 위치는 아래입니다.



```text

G:\\OneDrive\\01\_울산오박사인테리어\\obaksa\_site

```



수정용 ZIP 파일명 형식은 아래를 따릅니다.



```text

obaksa-home\_chatgpt\_YYYYMMDD\_HHMM.zip

```



수정용 ZIP에는 아래 항목을 포함합니다.



```text

루트 HTML 파일

assets/css/

assets/js/

cases.html 관련 파일

images/cases 최신 1\~2개 폴더

SITE\_UPDATE\_GUIDE.md

```



수정용 ZIP에는 아래 항목을 제외합니다.



```text

.git/

.vscode/

node\_modules/

images/gallery/

오래된 images/cases 전체

\*.zip

\*.bak

\*.tmp

\*.log

```



\## 8-3. ZIP 생성 후 확인 규칙



ZIP 생성 후에는 압축을 풀어 아래를 확인합니다.



```text

.git 폴더가 없는가

.vscode 폴더가 없는가

images/gallery가 빠졌는가

최근 images/cases 1\~2개만 들어갔는가

index.html이 있는가

cases.html이 있는가

assets/css가 있는가

assets/js가 있는가

SITE\_UPDATE\_GUIDE.md가 있는가

```

\# 9. 삭제하면 안 되는 파일



아래 파일과 폴더는 함부로 삭제하지 않습니다.



```text

.git/

.gitignore

index.html

about.html

cases.html

community.html

contact.html

assets/

assets/css/

assets/js/

images/

images/cases/

images/gallery/

make-release.ps1

make-chatgpt-package.bat

SITE\_UPDATE\_GUIDE.md

```



\## 9-1. .git 폴더



`.git` 폴더는 로컬 Git 관리용입니다.



삭제하면 Git 이력 관리가 깨질 수 있습니다.



배포 ZIP과 수정용 ZIP에서는 제외하지만, 로컬 프로젝트에서는 삭제하지 않습니다.



\## 9-2. .gitignore



`.gitignore`는 Git 관리에서 제외할 파일을 정하는 설정 파일입니다.



배포 ZIP에는 들어가지 않아도 되지만 로컬 프로젝트에는 유지합니다.



\## 9-3. images/gallery



`images/gallery`는 수정용 ZIP에서는 제외하지만 삭제하지 않습니다.



실제 홈페이지에서 참조 중일 수 있습니다.



\## 9-4. 오래된 시공사례 이미지



오래된 이미지가 당장 수정용 ZIP에 필요 없더라도 홈페이지에서 사용 중일 수 있습니다.



사용 여부 확인 없이 삭제하지 않습니다.



\## 9-5. 삭제 가능 후보



아래 항목은 확인 후 삭제할 수 있습니다.



```text

오래된 obaksa-home\_release\_\*.zip

오래된 obaksa-home\_chatgpt\_\*.zip

\*.bak

\*.tmp

\*.log

.DS\_Store

\_\_MACOSX

```



단, 삭제 전에는 목록을 먼저 출력하고 확인 후 삭제합니다.
